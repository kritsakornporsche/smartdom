# SmartDom: Systems Analysis & Overview

แอปพลิเคชัน **SmartDom** เป็นแพลตฟอร์ม SaaS (Software as a Service) สำหรับบริหารจัดการหอพักแบบครบวงจร ที่รองรับผู้ใช้งานหลายระดับ (Multi-tenant) จากการวิเคราะห์โครงสร้างโฟลเดอร์ (`app/api` และ `app/`), ฐานข้อมูล และการเชื่อมต่อของระบบ ผมสามารถจำแนกระบบย่อย (Subsystems) ทั้งหมดออกเป็น 8 ระบบหลักดังนี้ครับ:

---

## 1. Authentication & Authorization System (ระบบยืนยันตัวตนและจัดการสิทธิ์)
- **คำอธิบาย:** ระบบรักษาความปลอดภัยหลักของแอปพลิเคชัน ทำหน้าที่ตรวจสอบว่าใครคือผู้ใช้งาน และมีสิทธิ์เข้าถึงหน้าจอไหนบ้าง
- **เทคโนโลยีที่ใช้:** NextAuth.js (Auth.js) ร่วมกับ JWT
- **ฟีเจอร์หลัก:**
  - รองรับการล็อกอินผ่าน Email/Password (Bcrypt) และ Social Login (Google, Facebook, LINE ฯลฯ)
  - **RBAC (Role-Based Access Control):** แบ่งสิทธิ์ผู้ใช้ออกเป็น 5 ระดับ ได้แก่ `platform_admin` (แอดมินระบบส่วนกลาง), `owner` (เจ้าของหอพัก), `tenant` (ผู้เช่า), `keeper` (พนักงาน/แม่บ้าน), และ `guest` (บุคคลทั่วไป)
  - มี Middleware (`proxy.ts`) ที่คอยดักจับและเปลี่ยนหน้า (Redirect) ผู้ใช้ที่ไม่ได้รับอนุญาต

## 2. Dormitory Management System (ระบบจัดการข้อมูลหอพักหลัก)
- **คำอธิบาย:** ระบบที่เจ้าของหอพัก (`owner`) ใช้สำหรับบริหารจัดการทรัพยากรหลักของหอพักตัวเอง
- **ฟีเจอร์หลัก:**
  - **Room Management (`/api/rooms`):** เพิ่ม ลบ แก้ไข ข้อมูลห้องพัก, ระบุประเภทห้อง, ชั้น, และราคา
  - **Dormitory Profile:** ตั้งค่าข้อมูลพื้นฐานของหอพัก เช่น ค่าน้ำ ค่าไฟ สิ่งอำนวยความสะดวก เพื่อให้บุคคลภายนอก (Guest) เข้ามาดูได้
  - **Rules & Settings:** จัดการกฎระเบียบของหอพัก

## 3. Tenant & Contract Management System (ระบบบริหารจัดการผู้เช่าและสัญญา)
- **คำอธิบาย:** ดูแลวงจรชีวิตของผู้เช่าตั้งแต่ย้ายเข้าจนถึงย้ายออก
- **ฟีเจอร์หลัก:**
  - **Onboarding (`/api/owner/onboarding`):** จับคู่ผู้ใช้งาน (User) เข้ากับห้องพัก (Room) เพื่อเปลี่ยนสถานะเป็นผู้เช่า (Tenant)
  - **E-Contracts (`/api/contract`):** การทำสัญญาเช่าแบบดิจิทัล (Digital Signature) รองรับการเก็บลายเซ็นของทั้งผู้เช่าและเจ้าของ
  - **Move-out Requests:** ระบบสำหรับให้ผู้เช่ายื่นคำร้องขอย้ายออก และให้เจ้าของอนุมัติ

## 4. Billing & Accounting System (ระบบการเงินและบัญชี)
- **คำอธิบาย:** จัดการเรื่องรายรับ-รายจ่ายของหอพักแต่ละแห่ง
- **ฟีเจอร์หลัก:**
  - **Billing (`/api/owner/billing`):** ออกบิลค่าเช่ารายเดือนอัตโนมัติ (รวมค่าน้ำ/ไฟ) ผู้เช่าสามารถดูบิลผ่านแอปและอัปโหลดสลิปโอนเงิน (Slip URL) ได้
  - **Accounting (`/api/owner/accounting`):** ระบบบัญชีหลังบ้านของเจ้าของหอพัก สรุปรายรับ-รายจ่าย (Income/Expense) แบบรายเดือน 
  - **Stats (`/api/owner/stats`):** กราฟและสถิติภาพรวมเพื่อช่วยในการตัดสินใจทางธุรกิจ

## 5. Maintenance & Keeper System (ระบบแจ้งซ่อมและจัดการบุคลากร)
- **คำอธิบาย:** ระบบที่เชื่อมโยงผู้เช่า เจ้าของ และพนักงาน (Keeper) เข้าด้วยกัน
- **ฟีเจอร์หลัก:**
  - **Tenant Portal:** ผู้เช่าสามารถกดแจ้งซ่อม (Maintenance) พร้อมแนบรูปถ่ายปัญหา
  - **Job Assignment:** เจ้าของหอพักสามารถจ่ายงานให้กับพนักงานที่รับผิดชอบได้
  - **Keeper App (`/api/keeper/maid` & `/technician`):** หน้าจอเฉพาะสำหรับแม่บ้านและช่างซ่อม เพื่อกดรับงาน อัปเดตสถานะ (In Progress, Completed) และแนบรูปภาพหลังทำงานเสร็จ

## 6. Communication System (ระบบสื่อสารภายใน)
- **คำอธิบาย:** ช่องทางการสื่อสารระหว่างเจ้าของหอพักและผู้เช่า
- **ฟีเจอร์หลัก:**
  - **Announcements (`/api/announcements`):** เจ้าของสามารถประกาศข่าวสารให้ผู้เช่าทุกคนทราบ (เช่น แจ้งตัดไฟ) และระบบสามารถเช็คได้ว่าใครอ่านแล้วบ้าง (`announcement_reads`)
  - **Chat System (`/api/chat`):** ระบบแชท 1-ต่อ-1 ระหว่างผู้เช่าและเจ้าของหอพัก สำหรับสอบถามปัญหาโดยตรง

## 7. Platform Admin & SaaS Subscription System (ระบบบริหารจัดการส่วนกลาง)
- **คำอธิบาย:** ระบบระดับสูงสุด (Super Admin) สำหรับผู้สร้างซอฟต์แวร์ SmartDom ใช้บริหารจัดการเจ้าของหอพักที่มาใช้บริการแพลตฟอร์ม
- **ฟีเจอร์หลัก:**
  - **Packages Management (`/api/platform/packages`):** จัดการแพ็กเกจราคา (เช่น Starter, Pro) ควบคุมจำนวนห้องสูงสุดที่แต่ละหอพักสามารถสร้างได้
  - **Subscriptions (`/api/platform/subscriptions`):** ติดตามสถานะการสมัครใช้บริการ (Active, Expired) ของแต่ละหอพัก
  - **Platform Accounting:** บันทึกรายได้ของแพลตฟอร์ม (SaaS Revenue) ที่เก็บจากเจ้าของหอพัก

## 8. Booking & Guest System (ระบบการจองสำหรับบุคคลภายนอก)
- **คำอธิบาย:** ส่วนที่เปิดให้บุคคลทั่วไป (Guest) เข้ามาดูข้อมูลหอพักและทำการจอง
- **ฟีเจอร์หลัก:**
  - **Explore/Search:** ค้นหาหอพักที่อยู่ในระบบ SmartDom
  - **Booking Progress (`/api/booking`):** ผู้ใช้สามารถกดจองห้องพักล่วงหน้า (Booking) และสามารถติดตามสถานะการอนุมัติจากเจ้าของหอพักได้

---

### 💡 บทวิเคราะห์ทางสถาปัตยกรรม (Architectural Analysis)

- **ความครบถ้วน (Completeness):** โครงสร้างฟีเจอร์ของแอปพลิเคชันถือว่า **ครบถ้วนสมบูรณ์มาก** ครอบคลุมเกือบทุก Use Case ของธุรกิจหอพักและการทำ SaaS (Software as a Service)
- **การออกแบบ (Design Pattern):** ใช้การแยกโฟลเดอร์ตามบริบทของผู้ใช้ (Contextual Routing) เช่น `/owner`, `/tenant`, `/keeper`, `/platform` ซึ่งสอดคล้องกับ Role ของฐานข้อมูล ทำให้สเกลโปรเจกต์ได้ง่าย โค้ดไม่ปะปนกัน
- **จุดที่ต้องระวัง (Bottlenecks):** เนื่องจากทุกระบบทำงานอยู่บน Single Database (อ้างอิงจากการแก้ไขล่าสุด) ความท้าทายหลักคือการเขียน API ใน **ระบบที่ 2 ถึง 6** ที่จะต้องเคร่งครัดเรื่องการแนบเงื่อนไข `WHERE dorm_id = ?` เสมอ เพื่อไม่ให้ข้อมูลรั่วไหลข้ามหอพัก
