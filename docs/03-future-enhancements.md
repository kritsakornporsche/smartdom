# แผนการพัฒนาฟีเจอร์เพิ่มเติม (Free-Tier & No Cost)

ตามที่คุณต้องการให้พัฒนาระบบเพิ่มเติมโดยเน้นเฉพาะส่วนที่ **ไม่มีค่าใช้จ่าย (No Cost)** ผมได้คัดกรองฟีเจอร์ที่สามารถสร้างขึ้นมาได้ด้วยการเขียนโค้ดและปรับฐานข้อมูล โดยไม่ต้องพึ่งพา API หรือบริการภายนอกที่เสียเงินครับ

ฟีเจอร์เหล่านี้จะช่วยยกระดับระบบให้เป็น Property Management System (PMS) ระดับมืออาชีพมากขึ้น

## 📑 Proposed Features (สิ่งที่จะพัฒนา)

### 1. ระบบจัดการมิเตอร์น้ำ-ไฟ (Meter Management)
- **Database:** สร้างตาราง `meter_readings` เพื่อเก็บประวัติการจดมิเตอร์ (เลขเก่า-เลขใหม่) ของแต่ละห้อง
- **Logic:** เพิ่มหน้าจอให้เจ้าของหอพักกรอกเลขมิเตอร์ และระบบจะไปดึง Rate ค่าน้ำ/ไฟจาก `dormitory_profile` มาคำนวณยอดเงินของบิลเดือนนั้นให้อัตโนมัติ

### 2. ระบบสร้าง QR Code PromptPay (QR Code Generator)
- **Logic:** ใช้ Library `promptpay-qr` (ฟรี) ในการสร้างรูป QR Code สำหรับชำระเงิน
- **Flow:** เจ้าของหอพักเพิ่มเบอร์ PromptPay ลงในโปรไฟล์ -> เมื่อระบบออกบิล ผู้เช่าจะเห็น QR Code ที่ระบุยอดเงินเป๊ะๆ ให้สแกนจ่ายได้ทันที (ผู้เช่ายังต้องอัปโหลดสลิปยืนยันอยู่ แต่สะดวกขึ้นมาก)

### 3. ระบบแจ้งเตือนในแอป (In-App Notification System)
- **Database:** สร้างตาราง `notifications` เพื่อเก็บข้อความแจ้งเตือนของแต่ละ User
- **UI/UX:** เพิ่มไอคอน "กระดิ่งแจ้งเตือน" ที่แถบเมนู (Navbar) หากมีบิลใหม่, พัสดุมาถึง, หรือการเปลี่ยนสถานะแจ้งซ่อม ระบบจะเด้งแจ้งเตือนแบบ Real-time โดยไม่ต้องเสียค่าบริการส่ง SMS หรือ LINE

### 4. ระบบส่งออกเอกสารบัญชี (Data Export - CSV)
- **Logic:** เพิ่มฟังก์ชันในหน้า `Accounting` ให้เจ้าของหอพักสามารถกด "Export as CSV" เพื่อดาวน์โหลดรายรับ-รายจ่ายไปเปิดใน Excel ได้ โดยทำงานบนฝั่ง Client-side (ฟรี 100%)

### 5. ระบบจัดการทรัพย์สินในห้องพัก (Inventory Management)
- **Database:** สร้างตาราง `room_inventory` เพื่อจดบันทึกทรัพย์สิน (เช่น แอร์ โทรทัศน์ โต๊ะ) และสภาพปัจจุบัน
- **UI/UX:** เพิ่มหน้าย่อยในเมนู "ห้องพัก" ให้เจ้าของสามารถเช็กสภาพตอนผู้เช่าย้ายเข้า (Check-in) และย้ายออก (Check-out)

## 🛠️ Database Changes (ส่วนที่ต้องเพิ่มตาราง)

#### [NEW] `meter_readings`
- `id`, `dorm_id`, `room_id`, `type` (Water/Electric), `previous_reading`, `current_reading`, `billing_cycle`, `created_at`

#### [NEW] `notifications`
- `id`, `user_id`, `title`, `message`, `link_url`, `is_read`, `created_at`

#### [NEW] `room_inventory`
- `id`, `dorm_id`, `room_id`, `item_name`, `condition` (Good, Damaged, Replaced), `notes`, `created_at`

#### [MODIFY] `dormitory_profile`
- เพิ่มคอลัมน์ `promptpay_number` (VARCHAR) เพื่อใช้สร้าง QR Code
