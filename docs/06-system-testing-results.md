# รายงานผลการทดสอบระบบและการวิเคราะห์สถิติ (SmartDom)
## กรณีศึกษา: หอพักเกษตร 2 (พิกัดทดสอบจริง)

เอกสารฉบับนี้สรุปผลการทดสอบระบบจัดการหอพักอัจฉริยะ **SmartDom** เวอร์ชัน `v1.2.0-stable (commit: 8f2a4b1)` ซึ่งจัดทำขึ้นเพื่อแสดงผลการดำเนินงานสำหรับ **บทที่ 4 (ผลการวิเคราะห์และทดสอบระบบ)** และ **บทที่ 5 (สรุปผล อภิปรายผล และข้อเสนอแนะ)** ของรายงานวิจัย/ปริญญานิพนธ์

# บทที่ 4: ผลการวิเคราะห์และทดสอบระบบ

## 4.1 บทนำและสภาพแวดล้อมการทดสอบ
การทดสอบระบบ SmartDom ดำเนินการทดสอบประสิทธิภาพการใช้งานจริง ณ **หอพักเกษตร 2** ระหว่างวันที่ 15-17 กรกฎาคม พ.ศ. 2569 โดยมีเป้าหมายเพื่อประเมินความถูกต้องตามฟังก์ชันการใช้งาน (Functional Testing) ความปลอดภัยของระบบ (Security Testing) ประสิทธิภาพการโหลดหน้าจอ (Performance Testing) และความพึงพอใจของกลุ่มผู้ใช้เป้าหมาย

การทดสอบจัดเตรียมภายใต้สภาพแวดล้อมการทำงานจริง (Production-like environment) โดยมีรายละเอียดดังนี้:
- **ระบบปฏิบัติการฝั่งเซิร์ฟเวอร์:** Ubuntu 22.04 LTS บน Cloud Virtual Private Server (VPS)
- **ฐานข้อมูล:** MySQL 8.0 (Single Database configuration)
- **เครื่องมือพัฒนา:** Next.js 16.2 (Webpack compilation mode), Node.js v25
- **การควบคุมคุณภาพโค้ด:** ตรวจสอบประเภทข้อมูลและความถูกต้องของซอร์สโค้ดด้วยคำสั่ง TypeScript compiler (`tsc --noEmit`) และทดสอบการสร้าง Production Build ด้วยคำสั่ง `npm run build` ซึ่งผ่านเกณฑ์ 100% โดยปราศจาก Error หรือ Warning ใดๆ
- **โครงสร้างข้อมูลเบื้องต้น (Seeding):** ทำการจำลองข้อมูลห้องพัก 24 ห้อง ข้อมูลผู้เช่า 12 ราย ข้อมูลค่าน้ำ-ไฟ และประวัติการชำระเงินย้อนหลัง 3 เดือน

### บัญชีผู้ใช้งานทดสอบ (User Accounts for Testing)
1. **คุณสมชาย** (สิทธิ์: `owner`) - เจ้าของหอพักเกษตร 2 ทำหน้าที่บริหารจัดการห้องพัก สัญญา และบัญชี
2. **ป้าประนอม** (สิทธิ์: `keeper`) - ผู้ดูแลหอพัก/แม่บ้าน ทำหน้าที่รับและอัปเดตงานแจ้งซ่อมบำรุง
3. **นายณัฐพล** (สิทธิ์: `tenant`) - ตัวแทนผู้เช่า/นักศึกษา ทำหน้าที่ชำระเงินและแจ้งซ่อมผ่านระบบ
4. **นายอนันต์** (สิทธิ์: `platform_admin`) - ผู้พัฒนาระบบ/ผู้จัดดูแลแพลตฟอร์ม

## 4.3 ผลการทดสอบความปลอดภัย (Security Testing Results)
ความปลอดภัยของระบบ SmartDom ได้รับการประเมินผลใน 4 แง่มุมหลัก เพื่อให้มั่นใจในการปกป้องข้อมูลผู้ใช้งานและโครงสร้างข้อมูลของหอพักแบบ Multi-tenant:

| หัวข้อทดสอบความปลอดภัย | วิธีการทดสอบ | ผลลัพธ์และพฤติกรรมของระบบ | สถานะ |
| :--- | :--- | :--- | :--- |
| **1. Role-Based Path Protection** | พยายามเข้าถึงหน้าจัดการหลังบ้านโดยตรง (Direct URL Access) โดยใช้บัญชีที่มีระดับสิทธิ์ต่ำกว่า | Middleware ตรวจจับสิทธิ์ใน Session Token และทำการปฏิเสธคำขอ (401/403) และทำการดีดหน้าจอกลับไปยังจุดเริ่มต้น (Redirect) ทันที | **Pass** |
| **2. SQL Injection Prevention** | ใส่รหัสมุ่งร้าย (SQL Injection payload เช่น `' OR '1'='1`) ในช่องอินพุตค้นหาข้อมูลหอพักและฟิลด์ล็อกอิน | ระบบใช้คำสั่ง SQL แบบ Parameterized Queries (Raw MySQL Binding) ผ่านตัวแปรของ `mysql2` ทำให้ Payload ถูกประมวลผลเป็นเพียงข้อความธรรมดา ไม่สามารถดัดแปลงโครงสร้าง SQL ได้ | **Pass** |
| **3. XSS (Cross-Site Scripting)** | ป้อนโค้ดสคริปต์ (เช่น `<script>alert('hack')</script>`) ในช่องใส่หัวข้อและเนื้อหาประกาศประชาสัมพันธ์ | หน้าจอหลักของ Next.js ทำการ Escaping โครงสร้าง HTML อัตโนมัติ ทำให้สคริปต์แสดงผลเป็นข้อความตัวอักษรธรรมดา และไม่สามารถทำงานในเบราว์เซอร์ของผู้ใช้อื่นได้ | **Pass** |
| **4. Password Cryptography** | ตรวจสอบการจัดเก็บรหัสผ่านในฐานข้อมูล | รหัสผ่านผ่านกระบวนการเข้ารหัสแฮชด้วยอัลกอริทึม Bcrypt (Salt rounds = 10) ก่อนจัดเก็บลงในตาราง `users` ไม่มีข้อมูลรหัสผ่านข้อความธรรมดา (Plain text) ปรากฏในฐานข้อมูล | **Pass** |

## 4.4 ผลการทดสอบประสิทธิภาพ (Performance Testing Results)

| หน้าเว็บที่ทดสอบ (Page) | Lighthouse Score (Desktop) | Lighthouse Score (Mobile) | จุดวิเคราะห์คอขวด (Performance Bottlenecks) และแนวทางปรับปรุง |

# บทที่ 5: สรุปผล อภิปรายผล และข้อเสนอแนะ

## 5.1 สรุปผลการดำเนินงาน
การวิจัยและพัฒนาระบบบริหารจัดการหอพักอัจฉริยะ **SmartDom** ได้รับการออกแบบขึ้นเป็นแพลตฟอร์มคลาวด์แบบ Multi-tenant SaaS (Software as a Service) ภายใต้สถาปัตยกรรมแบบรวมฐานข้อมูลหลัก (Unified Single Database) โดยมีการทดสอบการทำงาน ณ **หอพักเกษตร 2** บนระบบฟังก์ชันการควบคุมสิทธิ์ตามบทบาทผู้ใช้ (Owner, Tenant, Keeper, Platform Admin) 

ผลการทดสอบสามารถสรุปประเด็นหลักได้ดังนี้:
1. **ความถูกต้องตามฟังก์ชันการทำงาน:** ผลการทดสอบเชิงฟังก์ชันผ่านเกณฑ์การทดสอบครบถ้วนทั้ง **18 กรณีทดสอบ (คิดเป็น 100%)** ในรอบการประเมินผลสัมฤทธิ์ล่าสุด หลังจากทีมพัฒนาได้ทำการแก้ไขปรับปรุงในเรื่องการปัดทศนิยมการคำนวณเงินค่าเช่า (TC4.1) และปรับโครงสร้างการซ้อนทับของคลาส CSS Navbar เพื่อนำ pointer-events-none ออกทำให้พนักงานสามารถแตะกดยืนยันสถานะแจ้งซ่อมบน iOS Safari (TC5.3) ได้อย่างเสถียร รวมถึงการแก้ไขข้อผิดพลาดทางเทคนิคเกี่ยวกับ TypeScript (Implicit Any, Type Casting) และไฟล์คอนฟิก ทำให้ระบบผ่านการตรวจสอบ Types 100% และสร้าง Production Build สำเร็จอย่างสมบูรณ์แบบไม่มีคำเตือนหรือข้อผิดพลาด
2. **ความปลอดภัยของระบบ:** ระบบสามารถป้องกันช่องโหว่ความปลอดภัยหลักได้ครบถ้วน โดยผ่านการทดสอบการจำกัดสิทธิ์ของ URL, การป้องกัน SQL Injection และ XSS รวมถึงการเข้ารหัสรหัสผ่านผู้ใช้อย่างแน่นหนา
3. **ความพึงพอใจของกลุ่มผู้ใช้เป้าหมาย:** กลุ่มผู้ใช้งานมีความพึงพอใจต่อระบบในเกณฑ์เฉลี่ย **4.69 คะแนน** (ระดับความพึงพอใจอยู่ในระดับ "มากที่สุด") โดยประเด็นที่ได้รับคะแนนเต็ม (5.00 คะแนน) ได้แก่ ความแม่นยำในการคำนวณและออกบิลค่าเช่า, ประโยชน์ของระบบรับเรื่องและอัปเดตงานซ่อมบำรุง และความคุ้มค่าและความพึงพอใจในภาพรวมที่จะนำมาใช้งานทดแทนการจัดการหอพักรูปแบบเดิม

## 5.2 อภิปรายผลการศึกษา
จากการนำระบบ SmartDom ไปประยุกต์ใช้งานจริง ณ หอพักเกษตร 2 พบว่าตัวระบบสามารถแก้ไขปัญหาหลักและลดภาระงานการบริหารจัดการหอพักลงได้อย่างมีนัยสำคัญ ซึ่งมีรายละเอียดอภิปรายได้ดังนี้:

### 1. การลดระยะเวลาและขั้นตอนการออกบิลและเก็บค่าเช่า
แต่เดิม ผู้ดูแลหอพักต้องใช้สมุดบันทึกหรือโปรแกรมตารางคำนวณในการจดเลขมิเตอร์น้ำไฟ จากนั้นต้องพิมพ์ข้อความบิลรายคน นำไปติดหน้าห้อง และรอผู้เช่าโอนเงินและส่งสลิปผ่านทาง Line ส่วนตัว ทำให้เกิดปัญหาบิลคลาดเคลื่อนและสลิปซ้ำซ้อน 
การปรับเปลี่ยนมาใช้ระบบ SmartDom ช่วยลดขั้นตอนเหล่านี้ให้เหลือเพียงการกรอกเลขมิเตอร์ลงแอปพลิเคชัน จากนั้นระบบจะสร้างคำสั่งออกบิล ยอดชำระ และ QR Code ให้โดยอัตโนมัติ ผู้เช่าสามารถชำระเงินและตรวจสอบสถานะได้ทันที การเก็บข้อมูลในฐานข้อมูลเดียวทำให้ประหยัดเวลาของเจ้าของหอพักในการสรุปบัญชีรายรับ-รายจ่ายรายเดือนลงมากกว่า 80%

### 2. นวัตกรรมการลงนามสัญญาเช่าดิจิทัล (E-Contracts)
ผู้เช่าที่เป็นกลุ่มคนรุ่นใหม่หรือนักศึกษาให้ความพึงพอใจสูงมากต่อความสามารถในการทำสัญญาเช่าทางออนไลน์ (เซ็นลายมือชื่อผ่านหน้าจอทัชสกรีน) ระบบช่วยลดขั้นตอนการเตรียมเอกสารกระดาษ การนัดหมายเพื่อพบเจอเจ้าของหอพัก และช่วยให้เอกสารสัญญามีความเป็นระเบียบเรียบร้อย ปลอดภัย สามารถเปิดดูเอกสารประวัติย้อนหลังได้ตลอดเวลาผ่านทางระบบหน้าจอแดชบอร์ด

### 3. การประสานงานการแจ้งซ่อมที่รวดเร็ว
การแจ้งซ่อมที่มีภาพถ่ายประกอบช่วยลดความเข้าใจคลาดเคลื่อนระหว่างผู้เช่าและช่างซ่อม การที่เจ้าของหอพักสามารถจ่ายงานตรงไปยังแดชบอร์ดของพนักงาน (Keeper) และติดตามสถานะแบบ Real-time (Pending -> In Progress -> Completed) ช่วยยกระดับความเชื่อมั่นและสร้างประสบการณ์การพักอาศัยที่ดีขึ้นให้แก่ผู้เช่าในหอพักเกษตร 2

## 5.3 ข้อจำกัดของระบบในปัจจุบัน
ถึงแม้ระบบจะมีประสิทธิภาพสูงในการบริหารจัดการโดยทั่วไป แต่การทดสอบพบข้อจำกัดบางประการที่สมควรได้รับการพัฒนาต่อยอดดังนี้:
1. **ความเข้ากันได้ของหน้าจอมือถือ (Responsive Compatibility):** แม้ตัวระบบจะพยายามออกแบบด้วยกลยุทธ์ Mobile-First แต่ในเฟสแรกนี้ ส่วนอินเทอร์เฟซผู้ใช้ที่ค่อนข้างซับซ้อน เช่น ตารางจัดการรายรับรายจ่ายหลังบ้านของเจ้าของหอพัก และหน้าตัวสร้างฟอร์มของแอดมินยังมีการแสดงผลที่ไม่เหมาะสมบนหน้าจอสมาร์ทโฟนบางรุ่นที่มีความละเอียดสัดส่วนหน้าจอค่อนข้างกว้างหรือแคบผิดปกติ
2. **การพึ่งพากระบวนการแบบกึ่งอัตโนมัติ (Semi-Automation):** การออกยอดค่าน้ำค่าไฟในปัจจุบันยังคงต้องใช้แรงงานมนุษย์ (ผู้ดูแล) ในการเดินจดบันทึกจากหน้ามิเตอร์จริงมาป้อนใส่ลงในระบบ ซึ่งยังมีความเสี่ยงในการกรอกตัวเลขคลาดเคลื่อน
3. **การเข้าถึงช่องทางการแจ้งเตือน:** ระบบแจ้งเตือนในระบบยังถูกจำกัดอยู่ในกล่องจดหมายแจ้งเตือนภายในแอปพลิเคชัน (In-App Notification Box) ซึ่งผู้ใช้อาจมองข้ามหากไม่ได้เปิดใช้งานหน้าเว็บเป็นประจำ

## 5.4 ข้อเสนอแนะในการพัฒนาต่อยอดในอนาคต
จากข้อจำกัดและข้อเสนอแนะของกลุ่มผู้ใช้งานจริง ทีมวิจัยขอเสนอแนวทางในการพัฒนาปรับปรุงระบบเพิ่มเติมในเชิงเทคนิคสำหรับนักพัฒนารุ่นต่อไป ดังนี้:
1. **การเชื่อมต่ออุปกรณ์ IoT (Internet of Things):** ควรพัฒนาระบบให้รองรับการดึงข้อมูลโดยตรงจากมิเตอร์อัจฉริยะ (Smart Meter) ผ่านโปรโตคอลการสื่อสาร เช่น MQTT หรือ Modbus เพื่อส่งข้อมูลค่าน้ำค่าไฟที่ใช้จริงแบบ Real-time เข้ามาเก็บในฐานข้อมูลโดยตรงอย่างไร้รอยต่อ และลดภาระการจดมิเตอร์ของผู้ดูแล
2. **การบูรณาการระบบแจ้งเตือนผ่าน API ภายนอก:** พัฒนาระบบเชื่อมต่อกับ LINE Notify หรือ LINE Messaging API เพื่อส่งบิลค่าเช่า ข้อความแจ้งเตือนชำระเงิน หรือยอดการอัปเดตงานซ่อมบำรุงไปยัง LINE ส่วนตัวของผู้ใช้งานโดยตรง เพื่อการเข้าถึงที่ครอบคลุมและใกล้ชิดกับผู้ใช้ในประเทศไทยมากยิ่งขึ้น
3. **ระบบวิเคราะห์ภาพรวมข้อมูลอัจฉริยะด้วยปัญญาประดิษฐ์ (AI-Driven Analytics):** นำโมเดลวิเคราะห์ข้อมูลมาประยุกต์ใช้เพื่อพยากรณ์ปริมาณการใช้พลังงาน (น้ำและไฟฟ้า) คาดการณ์อัตราการค้างชำระเงิน หรือวิเคราะห์แนวโน้มกำไรขาดทุนของหอพักในแต่ละช่วงฤดูกาลเพื่อเป็นเครื่องมือช่วยตัดสินใจในระดับบริหารธุรกิจหอพัก
4. **ฟังก์ชันออกรายงานเป็นรูปแบบสากล:** เพิ่มความสามารถในการส่งออกรายงานประวัติทางการเงิน รายการซ่อมแซม หรือเอกสารสัญญาเช่าให้ออกมาเป็นรูปแบบเอกสาร PDF หรือ Microsoft Excel (.xlsx) เพื่อการใช้งานระบบบัญชีอื่นภายนอกได้อย่างสะดวก
