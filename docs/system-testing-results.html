ผลการทดสอบการทำงานแยกตามกรณีทดสอบ (Functional Specification)

รหัส กรณีทดสอบ ผลลัพธ์คาดหวัง สถานะการทดสอบ ผลจริง / แนวทางการแก้ไข
TC1.1 ล็อกอินเข้าสู่ระบบด้วยบัญชีที่ถูกต้อง ระบบอนุญาตและเปลี่ยนเส้นทางไปยังหน้าหลักของสิทธิ์นั้น Pass เข้าสู่ระบบสำเร็จและนำไปยังหน้า Dashboard
TC1.2 ล็อกอินเข้าสู่ระบบด้วยบัญชีที่ผิดพลาด ระบบปฏิเสธการเข้าถึงและแสดงข้อความแจ้งเตือนข้อผิดพลาด Pass ปฏิเสธและแจ้งเตือนอย่างถูกต้อง
TC1.3 ตรวจสอบการจำกัดสิทธิ์หน้าจอตามบทบาท ผู้ใช้ไม่มีสิทธิ์เข้าถึงหน้าตั้งค่าของบทบาทอื่น Pass ดีดกลับ (Redirect) อัตโนมัติด้วย Middleware proxy.ts
TC2.1 เจ้าของเพิ่มประเภทห้องพักและราคา ข้อมูลถูกบันทึกลงตาราง rooms และแสดงในหน้าหลัก Pass บันทึกข้อมูลและอัปเดตห้องพักสำเร็จ
TC2.2 เจ้าของแก้ไขราคาห้องพักและสถานะ ข้อมูลห้องพักได้รับการปรับปรุงทันทีในฐานข้อมูล Pass ข้อมูลอัปเดตในระบบและตารางที่เชื่อมโยงทันที
TC2.3 เจ้าของลบข้อมูลห้องพัก ห้องพักที่ไม่มีผู้เช่าถูกลบออกจากระบบอย่างสมบูรณ์ Pass ลบได้ปกติและป้องกันห้องที่มีความสัมพันธ์กับสัญญาเช่า
TC2.4 เจ้าของกำหนดอัตราค่าน้ำและไฟฟ้า บันทึกอัตราค่าบริการลงในตาราง dormitory_profile Pass ตั้งค่าและบันทึกข้อมูลลงฐานข้อมูลเสร็จสิ้น
TC3.1 เจ้าของสร้างร่างสัญญาเช่าอิเล็กทรอนิกส์ ระบบสร้างรายการใหม่ลงตาราง contracts รอการอนุมัติ Pass ข้อมูลรายละเอียดสัญญาพร้อมสำหรับผู้เช่าเซ็นออนไลน์
TC3.2 ผู้เช่าลงนามสัญญาเช่าแบบดิจิทัล ลายเซ็นถูกแปลงเป็น Base64 และบันทึกเข้าระบบอย่างปลอดภัย Pass บันทึกลายเซ็นและอัปเดตสถานะการอนุมัติสัญญา
TC3.3 เจ้าของลงนามอนุมัติสัญญาเช่า สถานะสัญญาเปลี่ยนเป็น Active และผู้เช่าเป็น Tenant สมบูรณ์ Pass ทำรายการผ่านระบบและเชื่อมข้อมูลผู้เช่าและห้องพักโดยตรง
TC3.4 การจับคู่ผู้เช่ากับห้อง (Onboarding) ระบบเปลี่ยนสถานะห้องพักเป้น Occupied ทันทีหลังอนุมัติ Pass ผูกข้อมูลสำเร็จและแสดงผลได้อย่างถูกต้อง
TC3.5 ผู้เช่ายื่นคำร้องขอย้ายออก บันทึกคำร้องและแจ้งเตือนไปยังเจ้าของหอพักผ่านระบบ Pass ยื่นคำร้องสำเร็จและแสดงในหน้าจออนุมัติของเจ้าของ
TC4.1 การออกบิลค่าเช่ารายเดือนอัตโนมัติ คำนวณค่าน้ำ-ค่าไฟและสร้างบิลยอดชำระอัตโนมัติ Pass ปรับปรุงแก้ไขการปัดเศษในระบบหลังบ้านและกำหนดทศนิยมด้วย DECIMAL ร่วมกับ Math.round() ทำให้ยอดเงินตรง 100%
TC4.2 ผู้เช่าอัปโหลดสลิปโอนเงินชำระเงิน ภาพสลิปถูกเก็บใน Cloud Storage และบันทึก URL ลงในบิล Pass อัปโหลดและแสดงผลสลิปบนหน้าแดชบอร์ดเจ้าของหอพักทันที
TC4.3 เจ้าของหอพักกดยืนยันยอดโอนเงิน สถานะบิลเปลี่ยนเป็น Paid และระบบบันทึกเงินลงบัญชีรายรับ Pass ยืนยันสำเร็จและบันทึกยอดเงินเข้าพอร์ตหอพัก
TC4.4 เจ้าของเรียกดูบัญชีรายรับรายจ่าย สรุปภาพรวมรายรับรายจ่ายเป็นกราฟใน Dashboard Pass กราฟแดชบอร์ดสรุปสถิติการเงินได้อย่างแม่นยำและเรียลไทม์
TC5.1 ผู้เช่ายื่นแจ้งซ่อมบำรุงพร้อมรูปภาพ คำร้องถูกส่งไปยัง Backend พร้อมสร้างรายการแจ้งซ่อม Pass ระบบบันทึกและแสดงรายการปัญหาให้เจ้าของหอพักตรวจสอบ
TC5.2 เจ้าของมอบหมายงานให้ผู้ดูแล (Keeper) งานแจ้งซ่อมถูกจ่ายตรงไปยัง Dashboard ของ Keeper ที่ได้รับเลือก Pass จ่ายงานสำเร็จและอัปเดตสถานะเป็น In Progress
TC5.3 ผู้ดูแล (Keeper) อัปเดตสถานะเมื่อซ่อมเสร็จ แก้ไขสถานะผ่านหน้าจอ และแนบภาพเพื่อยืนยันงานเสร็จสิ้น Pass ปรับโครงสร้าง CSS Navbar เพื่อนำ pointer-events-none ออก ทำให้การกดปุ่มเปลี่ยนสถานะบน Safari (iOS) ทำงานเสถียร 100%

การทดสอบความปลอดภัยของข้อมูลระบบ (Security Testing)

Role-Based Protection Pass
วิธีทดสอบ: Direct URL Access

สกัดกั้นการเข้าถึงหลังบ้านโดยไม่มีสิทธิ์สำเร็จด้วย Next.js Middleware ที่ตรวจจับ Session Token และควบคุมการ Redirect กลับหน้าล็อกอินโดยอัตโนมัติ

SQL Injection Prevention Pass
วิธีทดสอบ: Parameterized payload inputs

ใช้ Parameterized Queries (Raw MySQL Binding) ผ่าน mysql2 ทำให้อินพุตแปลกปลอมไม่สามารถเปลี่ยนโครงสร้าง SQL และระบบมองเป็นเพียงสายอักขระธรรมดา

Cross-Site Scripting (XSS) Pass
วิธีทดสอบ: HTML script tag payloads

หน้าจอ React ใน Next.js มีกระบวนการ Escaping โครงสร้าง HTML อัตโนมัติ ป้องกันไม่ให้โค้ดมุ่งร้ายหรือสคริปต์ทำงานในเว็บบราวเซอร์ของบุคคลอื่นได้

Password Cryptography Pass
วิธีทดสอบ: DB Table Hashing Audit

รหัสผ่านทั้งหมดจะถูกแปลงเป็น Hash ผ่านไลบรารี Bcrypt (Salt rounds = 10) ก่อนบันทึกลงตาราง users ทำให้ไม่มีพาสเวิร์ดที่เป็นข้อความดิบรั่วไหลในระบบฐานข้อมูล
